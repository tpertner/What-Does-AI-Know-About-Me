# Contributing to What-Does-AI-Know-About-Me

Thank you for your interest in improving this toolkit! AI platforms change constantly, and the best version of this project is one that stays current and accurate through community contributions.

## What Kinds of Contributions Are Welcome

### 🟢 Highly Welcome

- **Platform updates** — New privacy settings, changed retention policies, new memory features
- **New platform coverage** — Adding audit prompts for Microsoft Copilot, Meta AI, Grok, DeepSeek, or others
- **Corrections** — Outdated screenshots, broken links, inaccurate facts, dead references
- **Prompt improvements** — Clearer phrasing, better structure, more effective audit questions
- **Real-world findings** — Interesting things you discovered while running the audits (anonymized)
- **Translations** — This toolkit should work for non-English speakers too
- **Accessibility improvements** — Better Markdown structure, clearer formatting, alt text for images

### 🟡 Welcome With Discussion First

Before starting work on these, please open an Issue to discuss:

- Major restructuring of the repo
- New sections or file types
- Changes to the core audit framework
- Stylistic rewrites of multiple files at once

### 🔴 Not a Fit for This Project

- **Jailbreak techniques** — This toolkit specifically avoids adversarial prompting. Contributions that push models to bypass safety systems will not be merged.
- **Commercial promotion** — Please don't add links to paid products or services
- **Unverified statistics** — Every factual claim needs a verifiable public source
- **Content that targets specific individuals or companies** with accusations
- **Code execution or scraping tools** — This is an educational/prompt toolkit, not a piece of software

## How to Contribute

### Small Changes (typo, broken link, factual update)

1. Fork the repo
2. Make your edit
3. Open a pull request with a clear title describing the change

### Larger Changes (new prompts, new platform, new guide)

1. **Open an Issue first** to discuss the idea
2. Wait for feedback before investing significant time
3. Fork the repo and create a branch with a descriptive name
4. Make your changes
5. Open a pull request referencing the Issue

## Quality Standards

All contributions should meet these standards:

- **Verifiable facts only.** Every statistic, policy claim, or privacy setting reference must have a citation to an official source (platform documentation, peer-reviewed research, reputable journalism, or primary legal documents).
- **Plain language.** Write for a mixed audience — executives, technical professionals, and everyday users should all be able to follow along.
- **Respectful framing.** This toolkit helps people, not attacks platforms. Keep the tone informative and professional.
- **Platform-neutral where possible.** Present comparisons fairly. Don't use the project to promote or attack any specific AI provider.
- **No PII.** Don't include personal information (yours or anyone else's) in examples or screenshots.
- **Date your sources.** AI platform policies change fast. Note the date you verified a claim, especially for anything in the platform comparison guide.

## Style Guide

- Use Markdown for all content files
- Use sentence case for headings (not Title Case)
- Use `—` (em-dash) for breaks in thought, not `--`
- Use fenced code blocks for prompts and commands
- Keep line lengths reasonable (no fixed limit, just don't write 2,000-character lines)
- Use relative links within the repo, not absolute GitHub URLs

## Review Process

Pull requests will be reviewed based on:

1. **Accuracy** — Is the content factually correct and verifiable?
2. **Clarity** — Is it easy to understand for the target audience?
3. **Alignment** — Does it fit the toolkit's purpose and tone?
4. **Quality** — Is it well-written and well-formatted?

You may be asked to make revisions before a PR is merged. This isn't personal — it's just part of maintaining quality.

## Questions?

Not sure if your contribution is a fit? Open an Issue and ask. No contribution is too small to discuss, and no question is too basic.

## Recognition

Contributors will be acknowledged in a CONTRIBUTORS file (once contributions start coming in). If you'd prefer to remain anonymous, let me know in your pull request.

Thank you for helping make AI data transparency accessible to more people.
